# PR修正コマンド

直前PR、または引数で指定されたPRのreview comment確認・修正対応実行。

## 実行
1. review comment確認
2. 問題点修正
3. コミット・プッシュ
4. github mcpまたはghでcopilotにreviewを再依頼

**方針**:
* 品質・可読性・セキュリティ向上、既存機能影響なし
* 指摘がすべて正しいとは限らない。修正前に仕様を調査し、実施の可否を判断すること
    * 未対応の場合はその理由をコメントに書き込む

## 作業完了報告（必須）

作業完了時は以下の情報を必ず報告すること：

### 報告テンプレート

```
PR修正対応が完了しました。

## 対応内容

### レビュー指摘への対応
- **対応した指摘数**: X件中Y件（Z%）
- **対応内容**:
  1. [指摘内容の要約] - ✅ 対応済み / ❌ 対応しない（理由）
  2. [指摘内容の要約] - ✅ 対応済み / ❌ 対応しない（理由）
  ...

### 修正内容の詳細
- **修正ファイル数**: N個
- **変更行数**: +追加行, -削除行
- **主な修正内容**:
  - [修正内容1]
  - [修正内容2]
  ...

### コミット情報
- **コミットハッシュ**: `abc1234`
- **コミットメッセージ**: "修正内容の概要"

### PR情報
- **PR URL**: https://github.com/owner/repo/pull/123
- **再レビュー依頼**: 完了 / 失敗（理由）

### 統計
- **対応率**: Y/X件（Z%）
- **所要時間**: 約N分
```
