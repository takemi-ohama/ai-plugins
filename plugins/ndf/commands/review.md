# PRレビューコマンド

直前PRを専門家としてレビュー。

## 実行
- 問題点・改善点を指摘→「Request Changes」
- 指摘なし→「Approve」

## 観点
コード品質・セキュリティ・可読性・保守性・テストカバレッジ

## 作業完了報告（必須）

作業完了時は以下の情報を必ず報告すること：

### 報告テンプレート

```
PRレビューが完了しました。

## レビュー情報

### 基本情報
- **PR URL**: https://github.com/owner/repo/pull/123
- **PRタイトル**: [PRのタイトル]
- **レビュー結果**: ✅ Approve / ⚠️ Request Changes / 💬 Comment
- **レビュー日時**: YYYY-MM-DD HH:MM:SS

### 変更内容の概要
- **変更ファイル数**: N個
- **変更行数**: +追加行, -削除行
- **変更の種類**: 新機能 / バグフィックス / リファクタリング / ドキュメント

### レビュー結果の詳細

#### ✅ 良い点
1. [良い点1]
2. [良い点2]
3. [良い点3]

#### ⚠️ 指摘事項（Request Changesの場合）
1. **[カテゴリ]**: [指摘内容]
   - **重要度**: Critical / High / Medium / Low
   - **ファイル**: `path/to/file.js:行番号`
   - **推奨対応**: [対応方法]

2. **[カテゴリ]**: [指摘内容]
   - **重要度**: Critical / High / Medium / Low
   - **ファイル**: `path/to/file.js:行番号`
   - **推奨対応**: [対応方法]

#### 💡 改善提案（オプション）

2. [改善提案2]

### レビュー観点別評価

| 観点 | 評価 | コメント |
|------|------|----------|
| コード品質 | ⭐⭐⭐⭐⭐ | [コメント] |

| 可読性 | ⭐⭐⭐⭐⭐ | [コメント] |
| 保守性 | ⭐⭐⭐⭐☆ | [コメント] |
| テストカバレッジ | ⭐⭐⭐☆☆ | [コメント] |

- **指摘した問題数**: N件

### 報告例（Request Changesの場合）

```
PRレビューが完了しました。

### 基本情報
- **PR URL**: https://github.com/takemi-ohama/ai-plugins/pull/13

- **レビュー結果**: ⚠️ Request Changes
- **レビュー日時**: 2026-01-31 07:00:00

- **変更ファイル数**: 7個
- **変更行数**: +275, -497
- **変更の種類**: 新機能（フックスクリプトの実装）

### レビュー結果の詳細

#### ✅ 良い点
1. Claude Code Hooks仕様に完全準拠した実装
2. コード削減（497行 → 275行、45%削減）
3. 実機能を持つ3つのスクリプトが正しく動作

#### ⚠️ 指摘事項
1. **セキュリティ**: AWS_SECRET_ACCESS_KEYの正規表現が広すぎる
   - **重要度**: Critical
   - **ファイル**: `plugins/affaan-m/hooks/secret-scan.js:15`
   - **推奨対応**: `/i`フラグを追加して厳密化

2. **ロジックエラー**: バイナリファイルのハンドリングがない
   - **重要度**: High
   - **ファイル**: `plugins/affaan-m/hooks/secret-scan.js:44`
   - **推奨対応**: バイナリ拡張子のフィルタリング追加

3. **ロジックエラー**: TypeScriptインストール確認がない
   - **重要度**: High
   - **ファイル**: `plugins/affaan-m/hooks/typescript-check.js:44`
   - **推奨対応**: `npx tsc --version`で事前確認

4. **ロジックエラー**: 複数lockファイルの競合処理
   - **重要度**: Medium
   - **ファイル**: `plugins/affaan-m/hooks/detect-package-manager.js:39`
   - **推奨対応**: 最新ファイルを優先する判定追加

#### 💡 改善提案
1. エラーメッセージをより詳細に記述
2. テストケースの追加検討

| 観点 | 評価 | コメント |
|------|------|----------|
| コード品質 | ⭐⭐⭐⭐☆ | 概ね良好だが、エラーハンドリングに改善余地 |
| セキュリティ | ⭐⭐⭐☆☆ | 正規表現の改善が必要 |
| 可読性 | ⭐⭐⭐⭐⭐ | コードは明確で理解しやすい |
| 保守性 | ⭐⭐⭐⭐⭐ | シンプルな構造で保守しやすい |
| テストカバレッジ | ⭐⭐⭐☆☆ | 手動テストのみ、自動テスト不足 |

### 統計
- **指摘した問題数**: 4件
  - Critical: 1件
  - High: 2件
  - Medium: 1件
  - Low: 0件
- **コメント数**: 4件
- **レビュー時間**: 約15分

### 次のアクション
